## Affected Pages/Routes

- `/brand/new` — brand profile creation form
- `/brand/[id]/edit` — brand profile edit form
- `/` — dashboard (redirect if no brand, project list, New Video modal)
- `/projects/[id]` — workflow view (auto-starts step 1, conversation thread per step)
- `POST /api/v1/projects` — creates project
- `POST /api/v1/projects/[id]/steps/[n]/generate` — LLM call (first gen or follow-up)
- `POST /api/v1/projects/[id]/steps/[n]/approve` — approves step, unlocks step N+1

## Key Interactions to Verify

- Create brand profile → save → redirect to dashboard → brand name visible in header
- Edit brand profile → save → next project load shows updated prompts in workflow view
- Click "New Video" → modal opens → fill title + pick character → submit → redirect to /projects/[id]
- Click "New Video" when no characters configured → skip picker → submit → redirect
- **Auto-start**: /projects/[id] page load → step 1 immediately starts generating (spinner shown, no click needed)
- Step 1 generates → response card shown with: Retry button, follow-up input, Approve button
- Click "Approve" on step 1 → step 1 turns green, step 2 becomes active (highlighted) with Generate button
- Click "Generate" on step 2 → step 2's LLM call fires → response shown
- Step 2's prompt includes step 1's approved output as context
- Add follow-up message on step 3 → "Make it shorter" → send → LLM responds with refinement (conversation continues)
- Click "Retry" on step 4 → conversation clears → fresh LLM call → new response shown
- Click approved step in sidebar → shows its approved response (read-only, no editing)
- External step (step 8) → instruction card + KlingAI link + asset URL input + Approve (no Generate)
- Complete all 10 steps → project shows "Completed" badge on dashboard

## Edge Cases

- New Video modal with 0 characters in brand profile → no character picker shown
- New Video modal with 1 character → auto-selected, no picker shown
- New Video modal with 2+ characters → radio list shown, selection required
- Brand profile form with invalid URL in logoUrl → inline validation error
- Approve with no LLM response (step never generated) → API returns 400
- Approve external step with empty outputAssetUrl → API returns 400
- Step 10 approve → project.status flips to 'completed'
- OPENAI_API_KEY missing or invalid → inline error on auto-start, Retry button shown
- LLM API rate limit hit → error shown inline, Retry available
- Double-click Generate → second click disabled while status = "generating"
- Follow-up message is empty string → send button disabled (no empty follow-up calls)
- Try to approve step 3 before step 2 is done → API returns 400
- Refresh page mid-generate (status = "generating") → page shows "Generating..." with Retry option
  (conversation[] has no assistant message yet, so UI shows error state + Retry)

## Critical Paths

- Brand profile setup (create once) → persists across sessions (refresh, data remains)
- Create Video → auto-generates step 1 → approve → step 2 active → generate → approve → ... → complete
- Step 1 system message: brand_name, brand_description, target_audience, tone, example_videos
- Step 2 system message: same brand context + "Step 1 output: [approved step 1 response]"
- Step 5 system message: brand context + steps 1-4 outputs + character context
- OPENAI_API_KEY is read from env server-side only — never exposed to client
- Follow-up conversation thread: original user message → assistant → "make it shorter" → assistant (refined)

## API Route Tests (M2)

`POST /api/v1/projects/[id]/steps/[n]/generate`

- First call (no followUpMessage): conversation = [user message]; calls LLM → assistant appended → 200
- Follow-up (followUpMessage = "make it shorter"): appends user msg to conversation → LLM called with full history → 200
- Retry (followUpMessage = undefined, retry: true): clears conversation → fresh call → 200
- Step 8-10 (external): returns 400 ("external step has no LLM call")
- Step N-1 not done: returns 400 ("complete step N-1 first")
- Step already "generating": returns 409 (prevents double-submit)
- LLM API error: returns 502, step status reset to "pending" (NOT "generating")
- Project not found: returns 404

`POST /api/v1/projects/[id]/steps/[n]/approve`

- Step is done (has llmResponse): returns 200, step.status = "done", step N+1 unlocked
- Step has no llmResponse (never generated): returns 400
- External step with empty outputAssetUrl: returns 400
- Already approved step: returns 200 (idempotent)
- Step N+1 does NOT auto-generate on approve (just unlocks) — verified by checking step N+1 status = "pending" after approve
- Step 10 approve → project.status flips to "completed"
